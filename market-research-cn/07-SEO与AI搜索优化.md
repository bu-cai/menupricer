# 第八部分：SEO ＋ AI搜索优化（GEO）完整打法

## 8.1 各渠道现状与策略（2026）

| 渠道 | 现状 | 策略要点 |
|---|---|---|
| **百度** | 广告收入连续下滑（2025Q2同比-15%），自身AI摘要（AI伙伴）截流首屏，但工具类「动作词」仍是最大入口；对新站考察期长（3-6月）；偏好备案域名、HTTPS、快速首屏 | ①必须ICP备案 ②百度站长平台主动推送+sitemap ③「动作词」优先（转/压缩/计算/生成）④落地页首屏即工具 ⑤小程序同步（百度智能小程序加权） |
| **Bing中国** | PC份额大涨（Edge默认+Windows捆绑），对新站/独立站友好度远高于百度，收录快 | 新站先吃Bing流量建立正反馈；Bing Webmaster+IndexNow |
| **360搜索** | 份额萎缩至个位数，但下沉/政企PC仍有存量；竞争小=容易排名 | 360站长平台提交即可，不专门优化，白捡流量 |
| **搜狗** | 并入腾讯后重心=微信搜一搜供给源之一 | 搜狗站长提交；重点其实在微信生态 |
| **微信搜一搜** | 月活8亿+量级，年轻用户「万物皆可搜一搜」；结果偏公众号文章+小程序+视频号 | ①公众号文章标题=精确长尾词（「2026年北京社保基数是多少」）②小程序名称卡品类词（搜一搜小程序权重高）③文章内嵌小程序卡片形成闭环 |
| **知乎搜索+站内SEO** | 知乎内容在百度权重极高（双吃：站内搜索+百度收录）；专业长文有长尾生命力 | 见第九部分专章 |
| **AI搜索（豆包/Kimi/元宝/DeepSeek/通义）** | 用户直接问AI要工具推荐；AI回答来源=搜索增强(RAG)抓取的网页+高权重UGC（知乎/CSDN/公众号） | 见8.5 GEO专章 |

## 8.2 关键词策略

### 关键词分层模型（以工资计算器为例）

| 层级 | 示例 | 竞争 | 承接页 |
|---|---|---|---|
| 品类大词 | 工资计算器、五险一金计算器 | 高 | 首页/主工具页 |
| 城市/参数长尾 | 2026杭州五险一金计算器、月薪2万上海到手多少 | 低 | pSEO城市页 |
| 问题词（知识型） | 五险一金交多少年、社保断缴有什么影响 | 中 | 工具页下方FAQ/博客 |
| 竞品/替代词 | XX计算器准吗、个税APP哪个好用 | 低 | 知乎回答承接 |
| 政策脉冲词 | 延迟退休政策2026、年终奖计税新规 | 时效 | 政策解读页(快速上线) |

### 选词流程
1. 5118/百度下拉+相关搜索挖长尾 → 2. 百度指数验证稳定性（避开纯脉冲词做主站）→ 3. 看SERP前10：若被知乎/百家号/烂页面占据=可打；若被大厂产品页占据=绕开 → 4. 一词一页，标题公式「{年份}{地区}{动作词} - {利益点}」。

## 8.3 程序化SEO（pSEO）规划

| 模板 | 变量 | 页面量 | 案例参考 |
|---|---|---|---|
| {城市}{年份}五险一金计算器 | 40城×年份 | 100+ | 各银行烂页面可轻松超越 |
| {考试}报名照片要求及在线处理 | 50考试×年份 | 100+ | 改图宝占位但质量低 |
| {城市}限行查询/{城市}尾号限行 | 20城×日更数据 | 20+动态 | APP强网页弱 |
| {格式A}转{格式B}在线 | 30格式两两组合 | 200+ | Aconvert模式 |
| {职业/场景}简历模板 | 200职业 | 200 | 锤子简历已验证 |
| {关系}称呼计算/怎么叫 | 100关系词 | 100 | 春节脉冲 |
| {省份}产假/育儿假天数 | 31省 | 31 | 几乎无竞争 |

**pSEO质量红线**（百度2024起严打低质聚合）：每页必须有真实差异化数据（该市基数/该考试规格）+可交互工具+独立FAQ，禁止只换地名的套壳页；页面间做「相邻城市/相关考试」内链网。

## 8.4 专题页与内容营销

- **专题页（Hub）**：「社保百科」「2026考试日历」「劳动维权工具箱」——聚合工具+文章，承接大词，向下分发权重。
- **内容日历跟政策走**：每年固定脉冲——1月(年终奖计税)、3月(汇算清缴)、6月(毕业论文/查重)、7月(社保基数调整)、9-10月(国考报名)、11月(个税专项扣除确认)、12月(年度总结/放假安排公布)。政策发布48小时内出「解读文+计算器更新」，抢时效窗口。
- **每个工具页配套3件内容**：怎么用(教程)、原理/政策依据(权威)、常见问题(FAQ Schema)。

## 8.5 GEO（AI搜索优化）专章

AI搜索（豆包/Kimi/元宝/DeepSeek联网模式）推荐工具的机制：**RAG检索高权重网页→摘要→附来源链接**。被引用的关键因素与做法：

| 因素 | 做法 |
|---|---|
| 内容可提取性 | 页面用「问题→直接答案→步骤→表格」结构；每个工具页首段用一句话说清「本工具是什么、解决什么、怎么用」（AI最爱抄首段） |
| 数据权威性 | 标注数据来源（人社局文号/国办发文号）+「最后更新：2026-XX」；AI偏好可验证的时效数据——**这正是计算器类的天然优势** |
| 结构化数据 | Schema.org：`WebApplication`(工具)、`FAQPage`、`HowTo`、`Dataset`(政策数据表)；虽然百度对Schema支持弱，但AI爬虫和Bing吃这套 |
| llms.txt | 站点根目录放llms.txt（站点说明+核心工具清单+数据更新频率），对国内AI爬虫是增量信号 |
| 高权重UGC提及 | AI训练与检索重度依赖知乎/CSDN/公众号——**在这些平台的高赞内容中自然提及你的工具=最强GEO**（比自站优化更有效） |
| 爬虫放行 | robots.txt允许主流AI爬虫UA；服务器不要屏蔽海外UA（Kimi等检索节点） |
| 品牌词一致性 | 全网统一品牌名+域名表述，让AI建立「XX工具→某域名」的实体关联 |
| 监测 | 每月人工问豆包/Kimi/元宝20个目标问题（「XX怎么算」「有什么工具推荐」），记录是否被引用，作为GEO北极星指标 |

## 8.6 技术SEO清单

| 项 | 要求 |
|---|---|
| 备案与主机 | ICP备案（百度收录几乎前提）；国内主机/CDN，首屏<1.5s |
| 渲染 | SSG/SSR（Next.js等），工具本体可以CSR但内容区必须服务端输出 |
| 移动端 | 百度移动优先；避免弹窗遮挡（百度落地页规范会降权） |
| 站长平台 | 百度(主动推送API)/Bing(IndexNow)/360/搜狗/神马 全部提交 |
| 内链 | 工具↔教程↔专题三层互链；面包屑 |
| 防采集 | 原创声明+版权页；百度原创保护提交 |
| 监测 | 百度统计+GSC(Bing)+5118排名监控；核心词周报 |
